# 🔎 星鉴流水线 v1.3 — Constitution-First Report Delivery with Gemini Scan, GPT Constitution, Claude Plan, and Independent Arbitration

```
┌─────────────────────────────────────────────────────────┐
│  🌐 Global State                                        │
│  ReportLevel: Q / S / D                                │
│  Drift: ALIGN / DRIFT / MAJOR_DRIFT                    │
│  Verdict: GO / REVISE / BLOCK                          │
└─────────────────────────────────────────────────────────┘

═══════════════════════════════════════════════════════════
  📥 Step 1 — 任务分级 + 报告类型判断
═══════════════════════════════════════════════════════════

  ☀️ main / opus

  1. 判断任务类型：GitHub 项目 / 外部方案 / 技术路线 / 产品化吸收
  2. 判断复杂度：Q / S / D
  3. 判断是否需要 Step 5 仲裁
  4. 发启动通知 → 监控群

═══════════════════════════════════════════════════════════
  🧭 Step 2A — Gemini 研究扫描
═══════════════════════════════════════════════════════════

  🧵 gemini

  输入：原始材料 / 仓库 / 约束
  产出：研究扫描
    • 核心问题
    • 边界
    • 假设
    • 风险
    • 路线候选
    • 需要由宪法定稿的项

  输出路径：`agents/gemini/reports/*-research-scan-*.md`
  通知：织梦群 + 监控群

═══════════════════════════════════════════════════════════
  🧭 Step 2B — GPT 研究宪法
═══════════════════════════════════════════════════════════

  🤖 openai (gpt-5.4)

  输入：Gemini 研究扫描 + 原始材料 / 约束
  产出：最终研究宪法
    • 目标
    • 非目标
    • 边界
    • 假设
    • 风险
    • 适配条件
    • 验收标准

  输出路径：`agents/openai/reports/*-constitution-*.md`
  通知：openai 群 + 监控群

═══════════════════════════════════════════════════════════
  🧠 Step 3 — Claude 主方案
═══════════════════════════════════════════════════════════

  🧑‍💼 claude (opus)

  输入：GPT 宪法
  产出：主方案 / 主报告
    • 推荐架构位置
    • Phase 0 / 1 / 2 计划
    • 立即启用项
    • 延后项
    • 默认配置基线
    • 风险与回滚触发器

  输出路径：`agents/claude/reports/*-plan-*.md`
  通知：小克群 + 监控群

═══════════════════════════════════════════════════════════
  ✅ Step 4 — Review/Gemini 一致性复核
═══════════════════════════════════════════════════════════

  🔎 review → 🧵 gemini

  基于最终研究宪法检查主方案是否跑偏
  输出：ALIGN / DRIFT / MAJOR_DRIFT

  ALIGN ✅           → Step 6
  DRIFT ⚠️          → 回 Step 3 修订一次
  MAJOR_DRIFT ❌     → Step 5

  通知：交叉审核群 + 监控群

═══════════════════════════════════════════════════════════
  ⚖️ Step 5 — Review/OpenAI 或 Claude 仲裁（按需）
═══════════════════════════════════════════════════════════

  🔎 review → 🤖 openai (gpt-5.4) / 🧑‍💼 claude (opus)

  仅在高风险 / 强分歧 / D 级时触发
  默认由 openai 仲裁；若 GPT 同源则切换 claude 独立仲裁
  按固定 rubric 仲裁，不自由发挥
  输入：宪法 + 计划 + Gemini 复核发现的问题
  输出：GO / REVISE / BLOCK

  通知：交叉审核群 + 监控群

═══════════════════════════════════════════════════════════
  📄 Step 6 — Docs 交付定稿
═══════════════════════════════════════════════════════════

  📚 docs

  统一：
    • 标题
    • 结论
    • 推荐路线
    • 风险
    • 报告路径
    • 交付摘要

═══════════════════════════════════════════════════════════
  📦 Step 7 — main 交付与通知
═══════════════════════════════════════════════════════════

  ☀️ main

  交付对象：晨星 + 监控群
  交付信号必须以：
    • 文件落地
    • 结构化回执
    • 通知已发/已补发
  三者共同成立

═══════════════════════════════════════════════════════════
  🏁 END
═══════════════════════════════════════════════════════════
```
