# 🔎 星鉴流水线 v1.0 — Constitution-First Report Delivery

```
┌─────────────────────────────────────────────────────────┐
│  🌐 Global State                                        │
│  ReportLevel: Q / S / D                                │
│  Drift: ALIGN / DRIFT / MAJOR_DRIFT                    │
│  Verdict: GO / REVISE / BLOCK                          │
└─────────────────────────────────────────────────────────┘

═══════════════════════════════════════════════════════════
  📥 Step 1 — 任务分级 + 报告类型判断
═══════════════════════════════════════════════════════════

  ☀️ main / opus

  1. 判断任务类型：GitHub 项目 / 外部方案 / 技术路线 / 产品化吸收
  2. 判断复杂度：Q / S / D
  3. 判断是否需要 Step 5 仲裁
  4. 发启动通知 → 监控群

═══════════════════════════════════════════════════════════
  🧭 Step 2 — Gemini 研究宪法
═══════════════════════════════════════════════════════════

  🧵 gemini

  输入：原始材料 / 仓库 / 约束
  产出：问题宪法
    • 目标
    • 边界
    • 假设
    • 风险
    • 适配条件
    • 不该做什么

  输出路径：`agents/gemini/reports/*-research-*.md`

═══════════════════════════════════════════════════════════
  🧠 Step 3 — review/opus 主方案
═══════════════════════════════════════════════════════════

  🔍 review / opus

  输入：Gemini 宪法
  产出：主方案 / 主报告
    • 推荐架构位置
    • Phase 0 / 1 / 2 计划
    • 立即启用项
    • 延后项
    • 默认配置基线
    • 风险与回滚触发器

  输出路径：方案 agent 自己目录

  说明：
     此处的 `review/opus` 承担主方案位
     不是星链里的代码审查位

═══════════════════════════════════════════════════════════
  ✅ Step 4 — Gemini 一致性复核
═══════════════════════════════════════════════════════════

  🧵 gemini

  检查主方案是否偏离研究宪法
  输出：ALIGN / DRIFT / MAJOR_DRIFT

  ALIGN ✅       → Step 6
  DRIFT ⚠️       → 回 Step 3 修订一次
  MAJOR_DRIFT ❌ → Step 5

═══════════════════════════════════════════════════════════
  ⚖️ Step 5 — GPT / Review 仲裁（按需）
═══════════════════════════════════════════════════════════

  🔍 review / GPT

  仅在高风险 / 强分歧 / D 级时触发
  输出：GO / REVISE / BLOCK

═══════════════════════════════════════════════════════════
  📄 Step 6 — Docs 交付定稿
═══════════════════════════════════════════════════════════

  📚 docs

  统一：
    • 标题
    • 结论
    • 推荐路线
    • 风险
    • 报告路径
    • 交付摘要

═══════════════════════════════════════════════════════════
  📦 Step 7 — main 交付与通知
═══════════════════════════════════════════════════════════

  ☀️ main

  交付对象：晨星 + 监控群
  交付信号必须以：
    • 文件落地
    • 结构化回执
    • 通知已发/已补发
  三者共同成立

═══════════════════════════════════════════════════════════
  🏁 END
═══════════════════════════════════════════════════════════
```
